# Дизайн: интерактивные эффекты site-v2 (мобайл + десктоп)

Дата: 2026-06-01
Статус: утверждён к реализации

## 1. Контекст

`site-v2` — одностраничное свадебное приглашение. Стек:
**React 19 + react-three-fiber (WebGL) + GSAP/ScrollTrigger + Lenis + Tailwind.**

Задача: добавить набор «вау»-эффектов, нацеленных в первую очередь на мобильную
версию, не ломая существующий функционал (intro-локскрин, desktop-hero,
RSVP-форма, countdown).

### Что уже есть в коде (фиксируем как данность)
- `HeroScene.tsx` — WebGL depth-параллакс по мыши (десктоп). Три ассета:
  цвет `main-lg.jpg`, depth `main-lg-depth-soft.png`, инпейнт «без людей»
  `main-lg-no-people.jpg`.
- `Scene.tsx` — `fixed inset-0 -z-10`, висит всю страницу. На тач-устройствах
  (`pointer: coarse`) WebGL **отключён**, показывается статичное фото
  `img_2309-hero` — сделано намеренно ради плавности нативного скролла.
- `SmoothScroll.tsx` — Lenis ↔ ScrollTrigger связаны корректно. **Lenis активен
  только на десктопе**; на тач — нативный скролл (иначе ломается схлопывание
  chrome в iOS Safari).
- `lib/gsap.ts` — зарегистрированы ScrollTrigger, DrawSVG, MorphSVG, SplitText.
  `Physics2DPlugin` есть в `node_modules`, но **не зарегистрирован**.
- Арки (`rounded-t-full border-olive`) уже есть в `Schedule` и `RSVP`, но без
  анимации раскрытия.
- `FilmGrain.tsx` — глобальное статичное зерно (opacity 0.08, multiply) поверх
  всего контента.
- `ColorSplash.tsx` — существует, но **нигде не подключён** (мёртвый код).
- `SectionPhoto.tsx` — адаптивное фото с опциональным scrub-параллаксом.
- Пайплайн ассетов: `generate-depth.mjs` (Depth Anything v2, локально),
  `generate-people-mask.mjs`. **Инпейнта «без людей» скрипта нет** — был сделан
  внешним AI.

## 2. Объём (что делаем) и анти-объём

### Делаем
1. Гиро-параллакс hero (полная depth-сцена) на мобайл; десктоп-мышь без изменений.
2. Pin & zoom — апгрейд существующего `PhotoBreak`.
3. Улучшенная арка (DrawSVG-обводка + раскрытие фото + внутренний параллакс) на `Schedule`.
4. Кино-оверлей (тёплые блики + лёгкое зерно) — только на фото арки `Schedule`.
5. Микро-секция «салют» (перебивка).
6. Микро-секция «сердечки» (перебивка).
7. Вибро-отклик (мобайл) — сопровождает салют и сердечки.

### НЕ делаем (зафиксировано с заказчиком)
- Отдельной секции «наша история» нет — структуру не переписываем.
- Стопка-карточек, горизонтальная галерея, scrub-перелистывание, scroll-reveal
  как отдельный эффект, оркестрованный вход, растущая лоза, декоративные
  разделители — исключены.
- Улучшение «нижних карточек» — исключено.
- Десктопный 3D-эффект слежения за мышью не меняем по поведению.

## 4. Общая архитектура

### 4.1. `useParallaxInput()` (новый хук)
Единый источник нормированного смещения `{x, y} ∈ [-1, 1]`.
- Десктоп (`pointer: fine`): слушает `pointermove`.
- Мобайл: слушает `deviceorientation` (`gamma/beta`, делёж ~30, кламп −1…1)
  **после** выданного разрешения.
- Сглаживание — `gsap.quickTo` (lerp).
- Потребители: hero-гиро; десктоп-детектор «потряхивания» в салюте может
  переиспользовать сырой `pointermove`.

### 4.2. `usePermissions()` (новый хук) + кнопка-приглашение
- На iOS 13+ запрашивает разом `DeviceOrientationEvent.requestPermission()` и
  `DeviceMotionEvent.requestPermission()`.
- Кнопка размещается в intro-локскрине (там уже есть тап-взаимодействие).
- Состояние (`granted/denied/unsupported`) — в контексте, его читают hero-гиро и
  салют.
- Где датчиков нет / отказ — эффекты тихо деградируют в статику.

### 4.3. GSAP / Lenis
- Дописать `Physics2DPlugin` в `gsap.registerPlugin(...)` в `lib/gsap.ts`.
- Все анимации — через `useGSAP({ scope })` (авто-чистка твинов и ScrollTrigger).
- Lenis не трогаем; на мобайл скролл-эффекты работают на нативном скролле через
  ScrollTrigger (pin/scrub поддерживаются).

### 4.4. WebGL-бюджет (критично для мобилки)
- На мобайл максимум один WebGL-момент на экране — это hero-гиро.
- Hero-canvas: `dpr={[1, 2]}`, `frameloop="demand"`, рендер только когда hero
  во вьюпорте (пауза по IntersectionObserver, инвалидация по input-кадрам).
- Кино-оверлей реализуем на CSS (не WebGL), чтобы не конкурировать с hero.

### 4.5. Доступность
- В каждом эффекте проверка `prefers-reduced-motion: reduce` →
  непрерывные/тяжёлые анимации выключены, контент в финальном видимом состоянии.

## 5. Спецификация по эффектам

### 5.1. Гиро-параллакс hero
**Ассеты:** выбрать вертикальное фото пары; сгенерить `depth-soft.png`
(`generate-depth.mjs`) и маску; **инпейнт «без людей» — отдельная ассет-задача**
(AI/ручная), без неё сцена не собирается.

**Код:** обобщить `HeroScene` — источник смещения берётся из `useParallaxInput`
(мышь ИЛИ гиро). В `Scene.tsx`: на тач, если разрешение выдано и устройство не
слабое и не reduced-motion — монтировать WebGL-сцену с `frameloop="demand"` +
IntersectionObserver-пауза; иначе оставить текущее статичное фото.

**Десктоп:** поведение слежения за мышью без изменений.
**Fallback:** нет гиро / reduced-motion / отказ → статичное фото (как сейчас).

### 5.2. Pin & zoom (апгрейд `PhotoBreak`)
**Код:** добавить `PhotoBreak` опциональный режим `pinZoom`. В этом режиме:
высокая секция (~260vh) с внутренним `position: sticky; height: 100svh`;
ScrollTrigger `pin: true, scrub: true`; таймлайн `scale 1.5 → 1.0`; подпись
fade-in после ~55% прогресса. Без флага `PhotoBreak` рендерится как сейчас.

**Fallback:** reduced-motion → статичный кадр в финальном масштабе (scale 1,
подпись видна).

### 5.3. Улучшенная арка (`Schedule`)
**Код:** обернуть арочное фото Schedule. Поверх — SVG-контур арки, рисуется
DrawSVG (`drawSVG: "0%" → "100%"`) по ScrollTrigger-появлению. Фото внутри
стартует со сдвигом/масштабом (`translateY` + `scale 1.1`, `opacity 0`) и
раскрывается (`clip-path`/`mask` + `y`/`scale`/`opacity`) при появлении.
Внутренний лёгкий scrub-параллакс фото — отдельный твин (переиспользовать
`SectionPhoto parallax`).

**Fallback:** reduced-motion → обводка и фото сразу в финальном виде.

### 5.4. Кино-оверлей (только фото арки Schedule)
**Код (CSS, лёгкий вариант):** два псевдо/слоя поверх фото —
(1) тёплые блики: радиальный градиент тёплых тонов, `mix-blend-mode: screen`,
медленный дрейф (CSS-анимация);
(2) лёгкое локальное зерно: SVG `feTurbulence`, `mix-blend-mode: overlay`,
низкая прозрачность.
Глобальный `FilmGrain` оставить тонким (без удвоения тяжёлого зерна).
`mix-blend-mode` — только на этом одном фото.

**Fallback:** reduced-motion → дрейф бликов и дрожание зерна выключены, слои
статичны.

### 5.5. Микро-секция «салют» (перебивка)
**Размещение:** между WhenWhere и Pin&zoom. Высота ~60–80vh, подсказка по центру.
**Частицы:** GSAP Physics2D (конфетти-DOM, гравитация/разлёт).
- Мобайл: `devicemotion`, порог по резкому скачку
  `accelerationIncludingGravity` (настраиваемый, ~32); вибро-паттерн.
- Десктоп: детект «потряхивания» курсором (высокая скорость `pointermove` +
  частые смены направления в коротком окне); запасной триггер — клик. Вибро нет.
- Подсказка адаптивная: «Тряхните телефон» / «Поводите мышкой».

**Fallback:** reduced-motion → без конфетти, обычная текстовая перебивка.

### 5.6. Микро-секция «сердечки» (перебивка)
**Размещение:** перед Finale (после Contacts). Высота ~60–80vh.
**Частицы:** GSAP-твины (разлёт + поворот + масштаб + fade), узлы удаляются.
- Мобайл: тап; вибро-импульс на касание.
- Десктоп: клик; дополнительно шлейф сердечек за курсором при движении
  (десктоп-эксклюзив).
- Переиспользовать/заменить мёртвый `ColorSplash`.

**Fallback:** reduced-motion → упрощённое появление без разлёта.

### 5.7. Вибро-отклик
`navigator.vibrate()` — короткий импульс на тап сердечек, паттерн на тряску.
iOS Safari игнорирует молча (ошибки нет).

## 6. Что на чём

- **WebGL (R3F):** только гиро-параллакс hero.
- **GSAP/ScrollTrigger:** pin&zoom, раскрытие арки (DrawSVG), частицы салюта
  (Physics2D) и сердечек.
- **CSS:** кино-оверлей, базовые состояния, формы арки.

## 7. Тестирование и защита от регрессий

После **каждого** эффекта:
- Прогон существующих e2e (`e2e/visual.spec.ts`).
- Smoke Playwright: intro проходится; hero виден; RSVP-форма сабмитит;
  countdown идёт; в консоли нет ошибок.

Новые Playwright-проверки:
- Pin&zoom: при скролле меняется масштаб; подпись появляется к концу.
- Арка: класс/состояние reveal навешивается при появлении.
- Reduced-motion: контент и фото видимы, тяжёлых анимаций нет.
- Микро-секции: клик / эмуляция события → частицы появляются и затем удаляются
  из DOM.
- Сенсоры: эмуляция через dispatch `DeviceOrientationEvent` / `DeviceMotionEvent`.

`tsc` + `vitest` — гейт перед коммитом (по правилам репозитория).
